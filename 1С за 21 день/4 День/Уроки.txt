Общая информация о регистрах
  Предназначение регистров
    Консолидировать информацию
    При необходимости быстро выдавать итоговый результат
  Информация в регистры попадает из документов
    В процессе проведения документов
  Схема информационной системы 1С
    Документ - Регистр - Отчет
------------------------------------------------------------------------------------------------------------------------------------------------
Регистры накопления
  Регистр накопления хранит информацию о показателях в определенное время
  Примеры регистров
    Остатки товаров
    Товары на складах
------------------------------------------------------------------------------------------------------------------------------------------------
Структура регистра
    Измерения - разрезы учета
    Ресурсы - показатели (числовые)
------------------------------------------------------------------------------------------------------------------------------------------------
Учет взаиморасчетов
  Задача
    Необходимо обеспечить учет взаиморасчетов
    Долги покупателей перед нами 
    Наши долги перед поставщиками
  Создание регистра "Взаиморасчеты"
    Определение структуры регистра
------------------------------------------------------------------------------------------------------------------------------------------------
Создаём новый регистр накопления "Взаиморасчёты" -> Вид регистра: остатки -> Данные -> Создаём новое измерение "Контрагент" -> Тип: СправочникСсылка.Контрагенты
Создаём новое ресурсы "Сумма" -> Тип: Число -> Длина: 15 -> Точность: 2
Регистраторы -> "ПоступлениеТоваровИУслуг" и "РеализацияТоваровИУслуг" отмечаем галочками
------------------------------------------------------------------------------------------------------------------------------------------------
Проведение документа
  Документ "Реализация товаров и услуг"
    Увеличивает долг покупателя
  Формирование движений в регистрах с помощью конструктора
  Движения документа "Поступление товаров и услуг"
  Просмотр записей регистра
------------------------------------------------------------------------------------------------------------------------------------------------
РеализацияТоваровИУслуг -> Движения -> Открываем регистры накопления и ставим галочку -> Конструктор движений -> Тип движения регистра: Приход -> Двойным кликом указываю на против контрагента Клиент -> Нажимаем "Ок"
Движения.Взаиморасчеты.Записывать = Истина;
	Движение = Движения.Взаиморасчеты.Добавить();
	Движение.ВидДвижения = ВидДвиженияНакопления.Приход;
	Движение.Период = Дата;
	Движение.Номенклатура = Клиент;

Создаём новый реквизит в "РеализацияТоваровИУслуг" и называем его "СуммаДокумента"-> Тип: число -> Длина: 15 -> Точность: 2
Конструктор движений -> Сумма   СуммаДокумента

Движения.Взаиморасчеты.Записывать = Истина;
	Движение = Движения.Взаиморасчеты.Добавить();
	Движение.ВидДвижения = ВидДвиженияНакопления.Приход;
	Движение.Период = Дата;
	Движение.Номенклатура = Клиент;
	Движение.Сумма = СуммаДокумента;

Формы -> Нажимаем на кнопку с изображением лупы -> Командный интерфейс -> Перейти -> Для регистра взаиморасчеты нужно поставить флажок видимости

F5

------------------------------------------------------------------------------------------------------------------------------------------------
Документ "Выписка банка"
  Отражает входящие и исходящие платежи по расчетному счету
  Табличные части
    Приход денежных средств
      Контрагент.Сумма
    Расход денежных средств
      Контрагент.Сумма
  Проведение документа
------------------------------------------------------------------------------------------------------------------------------------------------
Создаём документ "ВыпискаБанка" -> Данные -> создаем 2 табличные части "ПриходДенежныхСредств" и "РасходДенежныхСредств"

Создаём в каждой табличной части по 2 реквизита -> "Контрагент" и "Сумма"

Конструктор движений -> Расход -> Приход денежных средств -> Заполить выражения -> Приход -> Расход денежных средств -> Заполить выражения -> ОК
